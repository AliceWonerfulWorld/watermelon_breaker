# 要件定義書：音ゲー風・完全誘導ソナー（コードネーム：SuikaGroove）

本週末の「ハッカソン×運動会」において、目隠しをしたプレイヤーがスマホをポケットに入れた状態で、音とグルーヴ感だけでスイカに辿り着き、最高速度でターゲットを破壊するためのUX特化型Webアプリケーション。

---

## 1. 目的・ビジョン
* **脱・大声誘導**: 案内人が声を張り上げる従来のスイカ割りをハックし、プレイヤー単独の「没入型音ゲー体験」へと昇華させる。
* **画面レス完結**: スマホをポケット（運用想定）に入れたまま、画面を一切見ずに聴覚（ビート）と触覚（バイブレーション）だけで状況を完全把握する。
* **0秒通信・ローカル完結**: ネットワークトラブルを排除するため、すべてのセンサー処理・音声生成をブラウザ内部（フロントエンド）で完結させる。

---

## 2. ターゲットユーザーおよび運用想定
* **プレイヤー**: 目隠しをして、スマホをズボンのポケットに格納。
* **案内人（またはスタート時のプレイヤー）**: スタート地点でスイカのある方向へスマホを向け、基準方位をシステムにロックオンさせる。

---

## 3. 機能要件

### ① 方位キャリブレーション機能（ロックオン）
* **概要**: スイカのある絶対方向をアプリに記憶させる。
* **仕様**: 
  * スタート時、スマホの先端をスイカの方向へ向けた状態で「ターゲットロック」ボタンを押下する。
  * その瞬間のデバイスの向き（`alpha` 値など）を `0度（正解角）` として保存する。

### ② リアルタイム方位差分計算ロジック
* **概要**: 現在プレイヤーが向いている方角と、正解角のズレをリアルタイムに計算する。
* **仕様**:
  * `DeviceOrientationEvent` を監視。
  * 現在の角度と基準角の差分（`-180度` 〜 `+180度`）を常に算出する。

### ③ 聴覚ナビゲーション（Web Audio APIによる可変ビート）
* **概要**: 角度のズレ度合いに応じて、再生する音のテンポ・ピッチを滑らかに変調させる。
* **仕様**:
  * **正解範囲（ターゲット正面 ±15度以内）**: 
    * BGMが最高テンポになり、リズムゲームの「Perfect!」状態のような心地よい連続したビート（または効果音）が鳴り響く。
  * **許容範囲（±15度 〜 ±45度）**:
    * ターゲットに近づく（正面に近づく）ほど、テンポ（BGMのBPM）が徐々に加速する。
  * **迷子範囲（±45度以上）**:
    * テンポが著しく遅くなる、または不協和音・警告低音に切り替わり、方向が間違っていることを伝える。

### ④ 加速度センサーによる疑似距離測定（歩数検知）
* **概要**: プレイヤーの一歩を検知し、スイカとの想定距離を縮める。
* **仕様**:
  * `DeviceMotionEvent` の加速度（`accelerationIncludingGravity`）の閾値超えをトリガーに、歩数をカウント。
  * 初期設定された想定距離（例: 20歩）から、1歩歩くごとに残り距離を減算。
  * 残り歩数が減る（スイカに近づく）につれて、全体の音の「ピッチ（高低）」を上げる。

### ⑤ 到着・撃破トリガー（バイブレーション）
* **概要**: スイカの目の前に到達したことを伝える。
* **仕様**:
  * 残り距離（歩数）が0になった瞬間、BGMがファンファーレに変わり、スマホが激しく断続的にバイブレーション（`navigator.vibrate`）して、スイング（振り下ろし）の合図を送る。

---

## 4. 非機能要件・技術スタック

### ① フロントエンド・Web標準APIへの拘り
* 外部サーバーへの通信（WebSocketやREST API）は行わず、スタンドアロンで動作させる（通信ラグ0、オフライン動作可）。
* iOS/Androidのモバイルブラウザ（Safari/Chrome）で動作可能とする。

### ② 技術スタック
* **言語**: HTML5 / CSS3 / JavaScript (Vanilla JS)
* **主要API**:
  * `DeviceOrientation API`（コンパス・方位角）
  * `DeviceMotion API`（加速度・歩数）
  * `Web Audio API`（テンポ・ピッチのリアルタイム動的変調）
  * `Vibration API`（到着時の触覚フィードバック）

### ③ セキュリティ・本番環境要件
* モバイルブラウザで各種センサーおよび音声発音を有効化するため、**HTTPS環境でのホスティング**（GitHub Pages等）を必須とする。
* iOS等のブラウザ仕様に基づき、ユーザーの初回タップ（スタートボタン押下）をトリガーに AudioContext およびセンサーの利用許可（Permission）を取得する設計とする。

---

## 5. 画面UI構成（シンプルな1画面構成）
* **設定モード**: 
  * 「現在の向きをスイカに設定」ボタン
  * 想定歩数（初期距離）の設定スライダー
* **競技モード（ポケット内）**:
  * 現在のズレ角度、歩数のデバッグ表示（検証用）
  * 「リセット / 中止」ボタン（大きくタップしやすい領域）
